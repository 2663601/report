SenSE 论文详解

Semantic-Aware High-Fidelity Universal Speech Enhancement

西北工业大学 + 华为 | arXiv:2509.24708 | 2025年9月

1. 核心问题与创新点

1.1 解决的痛点

传统生成式语音增强方法（diffusion / flow matching）只在声学特征层面做映射（带噪 mel → 干净 mel），缺乏对高层语义信息的感知。这导致：

  • 在严重失真（强噪、丢包、强混响）下，语义模糊和声学不连续
  • 关键语音成分被噪声掩盖时，纯声学方法难以恢复
  • 而人类听重度污染语音时，会依赖语义先验（语境、词汇知识）来理解
核心创新：用语言模型（LM）显式建模语义，引导 flow-matching 生成过程。这是首次将"大语言模型范式"完整引入通用语音增强（USE）领域。

2. 整体架构（两阶段）

带噪语音
    |
    v
[Stage 1: SASLM 语义感知语音语言模型]
    - Whisper 编码器 → 连续 speech embedding
    - Transformer 自回归预测 → 净化的语义 token
    |
    v
[Stage 2: Flow Matching 语义引导增强]
    - 输入：带噪 mel + 语义 token + (可选) 参考 mel
    - DiT 主干 → 生成干净 mel
    |
    v
[BigVGAN 声码器 → 干净波形]

2.1 关键设计

  • 两阶段分离训练：先训语言模型，再训 flow matching
  • 语义 token 作为桥梁：连接语言模型和声学生成模型
  • 条件注入：语义+声学+参考 三重条件输入 DiT

3. 关键技术 1：SASLM 语义感知语音语言模型

组件具体配置作用
音频编码器Whisper-large-v3 (1280维) + adapter提取连续语音 embedding
语义分词器CosyVoice 𝒮³ Tokenizer v1 50Hz提取干净语音的语义 token（VQ/FSQ 量化）
语言模型Transformer (1536d, 20层, 16头)自回归预测净化语义 token

关键技巧

  1. 不量化 prefix：保留 encoder 输出的连续 embedding 作为 LM 输入前缀，避免量化损失
  2. 两阶段训练：先冻结 encoder 训练 LM，再冻结 LM 微调 encoder（让预训练的 ASR encoder 适应增强任务）
  3. 序列格式：<SOS>, speech_emb, <TOT>, speech_token, <EOS>
  4. 训练目标：标准 next-token prediction，loss 仅在 token 段计算

训练目标公式

p(x) = ∏ p(s_k | s_1, ..., s_(k-1), e_(1...n))
其中 s 为语义 token，e 为 speech embedding

4. 关键技术 2：Flow Matching 语义引导增强

4.1 Flow Matching 基础

Flow matching 学习一个速度场 v_θ，将先验分布（高斯噪声 x₀）流向目标分布（干净 mel x₁）：

x_t = (1-t)·x₀ + t·x₁,  t ∈ [0,1]
目标：v_θ((1-t)x₀ + t·x₁, conditions, t) ≈ (x₁ - x₀)

4.2 三重条件输入

条件来源作用
带噪 mel (y)输入语音的 mel 谱提供声学线索（频谱包络）
语义 token (z)SASLM 净化输出提供高层语义信息
参考 mel (可选)3秒目标说话人音频提供说话人特征

4.3 语义 token 与 mel 对齐

  • 语义 token (50Hz) 和 mel (~93Hz) 长度不一致
  • 用 filler token 填充对齐长度（参考 F5-TTS 思路）
  • 用 ConvNeXt V2 模块隐式对齐 token 和 mel 特征

4.4 主干网络

DiT (Diffusion Transformer)：1024 hidden, 22 layers, 16 attention heads

5. 核心训练技巧：Degrad Mask（降级掩码）

这是论文最重要的训练 trick

问题

训练时如果模型同时看到带噪语音和语义 token，会过度依赖带噪声学线索（更直接、更容易学），而忽略学习从语义 token 映射到干净语音的能力。

解决方案

训练时随机 mask 掉一部分带噪语音段，强迫模型学会从"语义 token + 周围上下文"重建被 mask 的部分：

m₁, m₂ = 二值时序掩码
模型输入：(1-t)x₀ + t·x₁, m₁⊙x₁ (masked clean), m₂⊙y (masked noisy), z (语义)
预测目标：(1-m₁)·(x₁ - x₀)  // 仅在 mask 区域计算 loss

效果（消融实验）

配置DNSMOSSpeechBERTScoreSIM-o
完整 SenSE3.4080.9160.837
w/o degrad mask下降下降初期更高，最终下降
w/o 语义引导显著下降显著下降显著下降

6. Prompt Guidance（说话人保持机制）

动机

严重失真下，带噪输入提供的说话人信息有限，纯增强容易丢失说话人特征。

实现

  • 推理时可选地输入 3 秒目标说话人参考音频
  • 通过 speech-infilling 任务训练实现：模型学会"补全"参考音频后的内容
  • 训练时随机选择参考片段，mask 掉其余部分

效果

测试集配置PESQSIM-o
DNS HardSet无 prompt2.0710.837
有 prompt2.2180.883
DNS Challenge GSR无 prompt2.2190.799
有 prompt2.4160.874

7.2 推理参数最优值

  • NFE = 8：flow matching 步数最优值（步数过多反而下降）
  • CFG = 0.5：classifier-free guidance 适中强度
  • Sway sampling = -1：在早期分配更多 step

8. SenSE vs URGENT Challenge 对比

8.1 相同点

维度共同点
任务定位都是 Universal Speech Enhancement（USE），处理多种失真
失真类型都覆盖噪声 + 混响 + 削波 + 带宽限制
数据模拟管道都通过混合 clean+noise+RIR 构造训练数据
评估指标都用 DNSMOS、NISQA、PESQ、ESTOI 等多维度评估
追求泛化性都强调单一模型处理多种退化

8.2 不同点（核心差异）

维度URGENT ChallengeSenSE
性质学术比赛（公开排行榜）单篇研究论文
测试集官方盲测集（含真实录音）自建测试集（DNS-HardSet, GSR）
方法路线各队伍自由探索（多阶段、Mamba、TF-GridNet等）专门聚焦 LM + Flow Matching
是否参赛各队伍正式提交评比未参加 URGENT，只对比 baseline
模型规模受比赛限制（一般 50-300M）700M+ 大模型
数据规模1300 小时（官方训练集）22000 小时（Emilia 数据集）
多语言URGENT 2025 强调多语言（5种）仅英语
主观评测包含 ITU-T P.808 真人 MOS仅客观指标
核心创新多样：多阶段级联、SSM、双路径引入 ASR 大模型 + 语义 token 引导

9. 独特价值与启示

9.1 SenSE 的独特价值

  1. 首次将大语言模型范式引入 USE：用 ASR 大模型（Whisper）+ 语义 tokenizer（𝒮³）完整接管语义通路
  2. 语义 token 不仅是辅助，还是显式条件：和带噪 mel 拼接进入 DiT，强制模型对齐
  3. Degrad Mask 训练策略：解决"模型偷懒"问题，通用于"多模态条件融合"场景
  4. Prompt Guidance：借鉴 TTS（F5-TTS）思路，把说话人保持和 SE 统一到一个框架

9.2 对比 URGENT 的启示

  1. "Scaling 大模型"路线 vs "精巧架构"路线：URGENT 多数队伍走精巧架构，SenSE 走"大数据+大模型+预训练编码器"的 LLM 路线
  2. 语义建模可能是下一个关键方向：URGENT 2024 lessons learned 指出"重叠语音、强噪、强混响"是最难场景——这些恰恰是需要语义先验的场景，SenSE 给出了一个解
  3. 生成式 vs 判别式权衡正在被打破：SenSE 的 dWER 反超判别式方法，说明语义引导能让生成式模型也保持高保真

9.3 局限性（论文自陈）

  • 噪声数据仅 700 小时（远少于干净语音的 22,000 小时），可能限制泛化
  • 模型参数量大（700M+）、计算开销高，不适合实时通信场景
  • 未参加 URGENT 等公开比赛，没有真人 MOS 主观评测对比
