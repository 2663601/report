1. 概述与设计思想

RNNoise 是由 Xiph.org 和 Mozilla 联合开发的实时语音降噪库。其核心创新在于采用 DSP + 深度学习混合架构：保留传统信号处理框架（FFT/IFFT、窗函数、频带划分、Pitch 滤波），仅用 RNN 替代最难调参的噪声估计和增益计算部分。

1.1 设计目标

  • 极低计算复杂度（树莓派 3 上 7x 实时，x86 上 60x 实时）
  • 极小模型体积（85KB，8-bit 量化权重）
  • 10ms 帧延迟，适合实时通信（WebRTC）
  • 无音乐噪声伪影

1.2 与传统方法和端到端方法的对比

特性RNNoise端到端 DNN传统 DSP (Speexdsp)
模型大小85KB~340KB数十 MB无模型
计算量极低需要 GPU极低
音质好最好一般
音乐噪声无可能有常见
调参难度低（训练即可）低极高（50%科学50%艺术）
延迟10ms通常更高10ms

3. 频带划分与增益机制

3.1 Bark 尺度频带

RNNoise 不直接操作每个频谱点，而是将频谱划分为若干临界频带（类 Bark 尺度）。低频频带窄（人耳分辨率高），高频频带宽（人耳分辨率低）。16kHz 版本使用 17-18 个频带。

3.2 频带增益

网络输出每个频带的增益 g[i]（范围 [0,1]），使用 sigmoid 激活函数天然保证范围。优势：

  1. 输出维度极低（18 vs 161 个频谱点）
  2. 整个频带统一操作，避免音乐噪声
  3. 不可能放大噪声（增益 <= 1）

4.3 GRU 门控机制

GRU 相比 LSTM 参数更少，包含两个门：更新门 z（控制保留多少历史状态）和重置门 r（控制是否忽略历史）。

z = sigmoid(W_z * [h_{t-1}, x_t])
r = sigmoid(W_r * [h_{t-1}, x_t])
h_candidate = tanh(W * [r * h_{t-1}, x_t])
h_t = (1-z) * h_{t-1} + z * h_candidate

5. Pitch 滤波（梳状滤波器）详细原理

5.1 背景：为什么需要 Pitch 滤波

频带增益机制的分辨率受限于频带宽度。以 16kHz 采样率为例，一个频带可能覆盖 200-400Hz 的范围。但语音的基频（pitch）通常在 80-500Hz，其谐波间距等于基频频率。

问题：当基频为 200Hz 时，谐波位于 200Hz, 400Hz, 600Hz...，谐波之间（如 300Hz）主要是噪声。但频带增益无法区分同一频带内的谐波和谐波间隙——要么整个频带通过，要么整个频带衰减。

解决方案：使用 Pitch 滤波器（梳状滤波器）在频带增益之前，先利用语音的周期性特征抑制谐波间的噪声。

5.2 梳状滤波器的数学原理

基本思想

语音信号具有准周期性，即 x(t) ≈ x(t - T)，其中 T 是 pitch 周期。噪声则没有这种周期性。如果我们将当前信号与 T 个采样点之前的信号做加权平均：

y(t) = x(t) + r * x(t - T)

其频率响应为：

H(f) = 1 + r * e^{-j2πfT}
|H(f)|² = 1 + r² + 2r*cos(2πfT)
  • 当 f = k/T（谐波频率）时：cos(2πk) = 1，|H| = 1 + r（增强）
  • 当 f = (k+0.5)/T（谐波间隙）时：cos(2π(k+0.5)) = -1，|H| = 1 - r（衰减）

这就形成了一个梳状滤波器——在谐波频率处通过，在谐波间隙处衰减，形状像梳子的齿。

频域实现

RNNoise 在频域实现 pitch 滤波，更加灵活：

// P[i] 是 pitch 周期偏移后的频谱（即 x(t-T) 的 FFT）
// X[i] 是当前帧频谱
// rf[i] 是滤波强度（每个频点不同）

X[i].r += rf[i] * P[i].r;
X[i].i += rf[i] * P[i].i;

等价于时域的：y(t) = x(t) + r * x(t - T)

5.3 Pitch 周期的估计

RNNoise 使用经典的自相关 pitch 检测算法：

  1. 下采样：将信号 2 倍下采样，减少计算量
  2. 自相关搜索：在 [PITCH_MIN_PERIOD, PITCH_MAX_PERIOD] 范围内搜索最大自相关
  3. 去倍频（remove_doubling）：防止 pitch 估计跳到 2 倍或 1/2 倍频率
// 16kHz 版本的 pitch 范围
#define PITCH_MIN_PERIOD 20   // 对应 800Hz (16000/20)
#define PITCH_MAX_PERIOD 256  // 对应 62.5Hz (16000/256)

5.4 Pitch 相关度（Voicing Strength）

pitch 相关度 Exp[i] 衡量每个频带中信号的周期性强度：

// 计算当前频谱 X 和 pitch 偏移频谱 P 的归一化互相关
compute_band_corr(Exp, X, P);
for (i=0; i<NB_BANDS; i++)
    Exp[i] = Exp[i] / sqrt(0.001 + Ex[i] * Ep[i]);
  • Exp ≈ 1：该频带高度周期性（清晰的语音谐波）
  • Exp ≈ 0：该频带无周期性（噪声或无声辅音）

5.5 滤波强度的自适应控制

pitch 滤波的关键是确定每个频带的滤波强度 r[i]。RNNoise 的策略：

for (i=0; i<NB_BANDS; i++) {
    if (Exp[i] > g[i])
        r[i] = 1;
    else
        r[i] = Exp[i]² * (1 - g[i]²) / (0.001 + g[i]² * (1 - Exp[i]²));
    r[i] = sqrt(clamp(r[i], 0, 1));
    r[i] *= sqrt(Ex[i] / (1e-8 + Ep[i]));  // 能量归一化
}
条件r[i] 值含义
Exp > g（相关度 > 增益）1（最强）信号周期性强且噪声大，全力滤波
Exp 高, g 低较大有语音谐波但噪声也大，积极滤波
Exp 低, g 高较小信号不太周期或噪声不大，轻微滤波
Exp 低, g 低中等噪声大但无明显谐波，适度滤波

5.6 能量归一化

pitch 滤波后信号能量会改变，需要归一化回原始能量：

// 计算滤波后的频带能量
compute_band_energy(newE, X);
// 归一化因子：保持原始能量不变
for (i=0; i<NB_BANDS; i++)
    norm[i] = sqrt(Ex[i] / (1e-8 + newE[i]));
// 应用归一化
interp_band_gain(normf, norm);
for (i=0; i<FREQ_SIZE; i++) {
    X[i].r *= normf[i];
    X[i].i *= normf[i];
}

这确保 pitch 滤波只改变频谱的"形状"（抑制谐波间噪声），不改变总能量。后续的频带增益才负责控制整体衰减。

5.7 物理直觉

想象一把梳子放在频谱上：梳齿对准语音谐波频率（通过），梳齿间隙对准噪声（衰减），梳齿间距 = 基频 = 1/pitch_period。

对于 200Hz 基频的男声，梳齿位于 200, 400, 600, 800... Hz，间隙在 100, 300, 500, 700... Hz。噪声均匀分布在所有频率，但语音能量集中在谐波上，所以梳状滤波能有效提升 SNR。

6.2 训练标签

  • 频带增益 g[i]：g = sqrt(Ey[i] / Ex[i])，即干净语音能量与带噪语音能量的比值开方
  • VAD 标签：基于干净语音帧能量的三值软标签（1.0/0.5/0.0）

7. 训练策略

7.1 数据增强（C 程序自动完成）

  • 随机混合语音和噪声文件
  • 随机 SNR（-5 ~ 15 dB）
  • 随机低通滤波（模拟不同带宽录音）
  • 10% 概率语音静音（纯噪声段）
  • 10% 概率噪声静音（纯语音段）

  • 每 2821 帧切换一次随机参数

7.2 损失函数

def mycost(y_true, y_pred):
    return mean(mymask(y_true) * (
        10 * (sqrt(y_pred) - sqrt(y_true))⁴  # 大误差强惩罚
        + (sqrt(y_pred) - sqrt(y_true))²      # 基础 MSE
        + 0.01 * binary_crossentropy(y_pred, y_true)  # 辅助项
    ))
  • sqrt 域计算：对小增益更敏感
  • 四次方项：惩罚严重的增益错误
  • mymask：忽略无效标签帧

7.3 权重约束

constraint = WeightClip(0.499)  # 所有权重限制在 [-0.499, 0.499]

便于 8-bit 定点量化部署。

8. 推理时的工程优化

8.1 增益平滑

float alpha = 0.6f;
g[i] = MAX(g[i], alpha * lastg[i]);

单向平滑：增益不能比上一帧的 60% 更低，避免突然衰减的"呼吸效应"。但允许快速上升（语音突然出现时不延迟）。

8.2 8-bit 量化

训练时权重约束在 [-0.5, 0.5]，推理时可用 8-bit 整数存储。原版模型 85KB（8-bit）vs 340KB（float32），无需浮点乘法，适合无 FPU 的嵌入式平台。

8.3 稀疏化

GRU 权重矩阵中大量接近零的值可以剪枝，进一步减少计算量。
