音频信号处理领域主要比赛调研

DNS Challenge / AEC Challenge / URGENT / CHiME / Clarity / L3DAS / VoiceMOS / SIG Challenge

概览

音频信号处理领域有多个国际性比赛，主要由 Microsoft、IEEE、ISCA 等机构组织，依托 ICASSP 和 Interspeech 两大顶会举办。这些比赛覆盖了从降噪、回声消除到语音质量评估的完整音频处理链路。

1. DNS Challenge（深度噪声抑制挑战赛）

Microsoft ICASSP/Interspeech 2020-2023

目标

推动实时单通道语音增强技术的发展，最大化增强语音的主观感知质量。重点关注深度学习方法在噪声抑制中的应用。

解决的问题

  • 实时通信中的背景噪声（办公室、街道、咖啡厅等）
  • 语音失真（过度抑制导致的语音损伤）
  • 混响和邻近说话人干扰
  • 个性化降噪（ICASSP 2023 新增：利用说话人注册音频区分目标和干扰语音）

关键特点

  • 实时约束：算法延迟不超过 40ms
  • 主观评价：使用 ITU-T P.835 标准（SIG/BAK/OVRL 三维度 MOS）
  • 大规模数据集：提供 750+ 小时干净语音 + 180+ 种噪声类型
  • 两个赛道：实时赛道（有计算约束）和非实时赛道

历届演进

届次会议新增重点
第1届Interspeech 2020基础单通道降噪
第2届ICASSP 2021宽带(48kHz)支持
第3届Interspeech 2021情感语音保护
第4届ICASSP 2022盲测集评估
第5届ICASSP 2023个性化降噪（说话人注册）

官方资源：GitHub

2. AEC Challenge（声学回声消除挑战赛）

Microsoft ICASSP 2021-2023

目标

推动声学回声消除技术的发展，解决全双工通信中的回声问题，同时保持近端语音质量。

解决的问题

  • 扬声器播放的远端信号通过麦克风回传（声学回声）
  • 非线性失真（扬声器饱和、功放失真）
  • 双讲场景下的回声消除（近端和远端同时说话）
  • 延迟估计和跟踪
  • 全频带(48kHz)回声消除

关键特点

  • 真实数据：提供真实设备录制的回声数据（非仿真）
  • 双讲评估：单独评估单讲和双讲场景
  • MOS + WER：同时评估语音质量和语音识别率
  • 个性化赛道（第4届）：利用说话人信息辅助回声消除

评估指标

  • 单讲场景：回声抑制后的 MOS
  • 双讲场景：近端语音保留质量 MOS
  • 语音识别率（WER）

官方资源：GitHub

3. URGENT Challenge（通用鲁棒语音增强挑战赛）

CMU/多机构 ICASSP/Interspeech 2024-2026

目标

推动通用、鲁棒、可泛化的语音增强系统，能够处理多种失真类型、多语言、多领域的输入条件。

解决的问题

  • 单一模型处理多种失真（噪声、混响、削波、带宽限制、丢包等 7 种失真类型）
  • 跨语言泛化（覆盖 5 种语言）
  • 数据可扩展性（使用带噪数据训练）
  • 语音质量评估（Track 2：增强语音的自动质量评估）

关键特点

  • 通用性：不针对单一任务，要求一个模型处理所有失真
  • 双赛道：Track 1 通用增强 + Track 2 质量评估
  • 大规模参与：80+ 团队注册，29 个有效提交（ICASSP 2026）
  • 多指标评估：PESQ、STOI、SI-SDR、DNSMOS 等综合评估

官方网站：urgent-challenge.com

4. CHiME Challenge（噪声环境语音处理挑战赛）

Sheffield大学等 独立Workshop 2011-2024

目标

推动鲁棒自动语音识别（ASR）技术，促进语音处理、信号处理和机器学习交叉领域的研究。

解决的问题

  • 真实噪声环境下的语音识别（公交车、咖啡厅、街道等）
  • 远场语音识别（麦克风阵列）
  • 多说话人重叠语音分离与识别
  • 会议场景的说话人日志和转录（CHiME-7/8 NOTSOFAR）

关键特点

  • 真实录音：使用真实环境录制的数据（非仿真）
  • 多通道：支持 2/4/6 通道麦克风阵列
  • 端到端评估：最终指标是 WER（词错误率）
  • 前后端联合：鼓励信号处理前端与 ASR 后端联合优化

历届演进

届次年份重点场景
CHiME-1/22011/2013家庭环境噪声下的关键词识别
CHiME-3/42015/2016平板电脑录音的真实噪声 ASR
CHiME-52018家庭聚会多说话人对话
CHiME-62020多阵列远场对话转录
CHiME-72023多场景多设备通用 ASR
CHiME-82024NOTSOFAR 会议室场景

官方网站：chimechallenge.org

5. Clarity Challenge（助听器语音清晰度挑战赛）

UK Clarity Project ICASSP/独立 2021-2025

目标

改善听力受损用户在日常噪声环境中的语音可懂度，推动助听器信号处理算法的机器学习化。

解决的问题

  • 听力受损者在噪声环境中的语音理解困难
  • 助听器的个性化信号处理（根据听力图定制）
  • 语音可懂度的自动预测（无需人工听力测试）
  • 多干扰源、头部运动等动态场景

两类子挑战

  • CEC（Enhancement Challenge）：设计助听器增强算法，最大化听障用户的语音可懂度
  • CPC（Prediction Challenge）：预测听障用户对增强语音的可懂度评分（无需真人测试）

关键特点

  • 听力图驱动：每个测试样本附带听者的听力损失曲线
  • 双耳处理：6 通道输入（每耳 3 个麦克风）
  • 真实评估：最终由真实听障用户进行听力测试

官方网站：claritychallenge.org

6. L3DAS Challenge（3D音频机器学习挑战赛）

Sapienza大学 ICASSP 2021-2023

目标

推动 3D 空间音频场景下的机器学习方法，包括 3D 语音增强和 3D 声音事件定位与检测。

解决的问题

  • Ambisonics（一阶环绕声）录音中的语音增强
  • 3D 空间中的声源定位（方位角、仰角、距离）
  • 声音事件检测（识别声音类别和时间边界）
  • 办公室等混响环境下的空间音频处理

关键特点

  • Ambisonics 格式：使用一阶 FOA（4通道）录音
  • 真实环境：在真实办公室中录制
  • 双任务：Task 1 语音增强（STOI指标）+ Task 2 声音事件定位检测
  • 1-mic 和 2-mic：支持单阵列和双阵列配置

官方网站：l3das.com

7. VoiceMOS Challenge（语音质量MOS预测挑战赛）

NVIDIA/NII Interspeech/SLT 2022-2024

目标

推动语音质量自动评估技术，预测人类对语音的主观评分（MOS），减少对昂贵人工听力测试的依赖。

解决的问题

  • 合成语音的自然度评估
  • 增强语音的质量评估（SIG/BAK/OVRL）
  • 歌声合成和语音转换的质量预测
  • 跨系统、跨语言的 MOS 预测泛化

关键特点

  • 多赛道：高质量合成语音 / 歌声 / 带噪增强语音
  • 无参考评估：不需要干净参考信号
  • 小样本：Track 3 仅提供 100 个标注样本训练
  • ITU-T P.835：评估 SIG（语音质量）、BAK（背景噪声）、OVRL（整体质量）

官方资源：voicemos-challenge.github.io

8. SIG Challenge（语音信号质量改善挑战赛）

Microsoft ICASSP 2023-2024

目标

提升通信系统中语音信号的整体质量，使用真实（非仿真）测试集评估实时算法的综合表现。

解决的问题

  • 通信系统中的综合语音质量问题（噪声+回声+混响+编解码失真）
  • 联合处理多种退化（不是单独解决某一个问题）
  • 真实设备和网络条件下的语音质量

关键特点

  • 真实测试集：使用真实通信场景录制的数据评估
  • 综合任务：同时处理 AEC + NS + 去混响
  • 实时约束：算法必须满足实时处理要求
  • P.835 评估：使用 ITU-T P.835 主观评估框架

官方资源：GitHub

比赛关系与技术栈对应

音频通信处理链路中各比赛的覆盖范围：

麦克风输入 → [AEC回声消除] → [DNS降噪] → [去混响] → [编码传输] → [解码] → [质量评估]

  • AEC Challenge：覆盖回声消除环节
  • DNS Challenge：覆盖噪声抑制环节
  • SIG Challenge：覆盖整条链路的联合优化
  • URGENT Challenge：通用增强（所有退化类型）
  • VoiceMOS Challenge：覆盖质量评估环节
  • CHiME Challenge：前端增强 + 后端 ASR 联合
  • Clarity Challenge：面向听障用户的特殊增强
  • L3DAS Challenge：3D 空间音频场景
