1. 算法模块使能标志总览

标志含义设置方式
aec_enable回声消除Create 时或 EZ_VQEM_SET_AEC_ENABLE
ns_enable传统降噪(WebRTC NS)Create 时或 EZ_VQEM_SET_NS_ENABLE
dse_enableAI 降噪(DFNET/GTCRN)EZ_VQEM_SET_DSE_ENABLE
hs_enable啸叫抑制YS_VQE_SET_HS_ENABLE
agc_enable自动增益控制Create 时或 EZ_VQEM_SET_AGC_ENABLE
hpf_enable高通滤波Create 时
vad_enable语音活动检测ALC/DSE 开启时自动开启
alc_enable自动电平控制Create 时
dse_levelAINS 模式SetConfig (0=DFNET, 1=人声增强, 2=GTCRN)

2. ProcessCore 处理流程

输入 → ALC(自动电平) → HPF(高通滤波) → 分频(48k→3子带) → 啸叫检测(YS_HowlSup_Process) → AEC(回声消除) → NS(传统降噪) → 合频(3子带→48k) → DSE/AINS(AI降噪) → AGC(自动增益) → VAD(语音活动检测+抑制) → 输出

关键设计：啸叫检测在 AEC 之前运行（基于原始输入信号），确保检测不受 AEC/AINS 处理影响。

3. AEC 跳过逻辑

3.1 判断变量

int hs_skip_aec = 0;
int hs_skip_aec_process = 0;

if (hs_enable == Enable) {
    if (dse_level == 1)           → hs_skip_aec = 1;  // 人声增强: 完全跳过
    else if (hs_howl_detected_ever) → 不跳过;          // 啸叫已检测: AEC 正常
    else if (!hs_coldstart_done)    → 不跳过;          // 冷启动期: AEC 正常
    else                            → hs_skip_aec = 1; // 冷启动过期+无啸叫: 跳过
}

3.2 条件汇总

条件hs_skip_aec说明
dse_level == 1（人声增强）跳过人声增强模式完全不走 AEC
啸叫已检测到不跳过AEC 正常运行
冷启动期内（前10秒）不跳过AEC 运行用于收敛判断
冷启动过期 + 无啸叫跳过判定不在同一空间

4. NS 跳过逻辑

int skip_ns = (hs_enable && dse_enable && howl_fade_counter > HOWL_WARMUP_FRAMES)
           || (dse_level == 1 && dse_enable);
条件跳过 NS原因
啸叫模式 + AINS 已完成 crossfade跳过AINS 已接管降噪，NS 冗余
人声增强模式 (dse_level==1)跳过GTCRN 已接管降噪
其他情况不跳过NS 正常运行

5. DSE(AINS) 触发逻辑

int run_dse = (dse_enable == Enable);
if (run_dse && hs_enable && dse_level != 1) {
    run_dse = (howl_fade_counter >= 0);  // 需要啸叫检测到才运行
}
条件运行 DSE说明
dse_enable + 无 hs_enable运行正常 AINS 模式
dse_level == 1（人声增强）始终运行不受啸叫检测影响
hs_enable + 啸叫已检测运行啸叫触发后 AINS 启动
hs_enable + 啸叫未检测不运行等待啸叫检测结果

6. 啸叫抑制详细逻辑

6.1 整体设计思路

啸叫抑制模块的核心思想：先用 AEC 自适应滤波器检测是否存在啸叫风险（设备在同一空间），再决定是否启用 AINS 进行深度降噪。

6.2 两种工作模式

模式条件行为
AEC + AINS 模式hs_enable + dse_enable啸叫检测 + AINS 联合工作
纯 AEC 模式hs_enable + !dse_enable仅用自适应滤波器做啸叫抑制

6.3 啸叫检测流程（冷启动期）

┌─────────────────────────────────────────────────────────────┐ │ 冷启动期（前10秒） │ │ │ │ 1. AEC 始终运行（BufferFarend + Process） │ │ 2. 啸叫检测模块（YS_HowlSup_Process）每帧运行 │ │ 3. AEC 收敛度检测（EvzAec_convergence_level） │ │ │ │ 触发条件（满足任一即判定啸叫）： │ │ a) YS_HowlSup_Process 返回 1（频域啸叫特征检测） │ │ b) AEC 收敛度 >= 2（强回声相关性 = 同一空间） │ │ │ │ 结果： │ │ - 检测到 → hs_howl_detected_ever = 1, 启动 AINS warm-up │ │ - 10秒内未检测到 → 冷启动过期，停止 AEC │ └─────────────────────────────────────────────────────────────┘

6.4 AINS 启动的 Fade 状态机

检测到啸叫后，AINS 不是立即切换，而是经过 warm-up + crossfade：

howl_fade_counter状态行为
-1正常（未检测到啸叫）AINS 不运行
0刚检测到开始 warm-up
1 ~ 2warm-up 帧AINS 运行但丢弃输出，使用 NS 信号
3 (HOWL_WARMUP_FRAMES)crossfade 帧NS 信号 fade-out + AINS 信号 fade-in
> 3正常 AINS 输出同时 skip NS

6.5 Crossfade 实现

// crossfade 帧：NS 信号渐出 + AINS 信号渐入
for (data_j = 0; data_j < block_len; data_j++) {
    float gain_in  = scale_table[data_j];           // 0→1 (AINS渐入)
    float gain_out = scale_table[len - 1 - data_j]; // 1→0 (NS渐出)
    tmp_buf[data_j] = (short)(
        howl_raw_save[data_j] * gain_out +
        tmp_buf[data_j] * gain_in);
}

6.6 远端信号延时缓冲

啸叫抑制使用独立的远端缓冲区（hs_aec_far_f），长度由视频分辨率和码流模式决定：

stream_mode分辨率hs_far_buf_len对应延时(约)
1（主码流）≥ 4K (3840×2176)70 帧~700ms
1（主码流）< 4K30 帧~300ms
2（子码流）> 2K (2560×1440)150 帧~1500ms
2（子码流）≤ 2K30 帧~300ms

原因：视频编解码引入的延时会导致音频回路延时增大，高分辨率/子码流延时更大。

6.7 关键数据结构

// VQE_HANDLE 中啸叫相关字段
void* hHS;                      // 啸叫检测模块句柄
int hs_hc_flag;                 // 当前帧啸叫检测结果 (1=啸叫, 0=正常)

float hs_aec_far_f_48k[200][480]; // 48k 远端历史缓冲

6.8 啸叫检测初始化 (YS_VQE_SET_HS_ENABLE)

  1. 创建 WebRTC AEC 实例（hAEC[MAX_CHANNELS]），nlp_level=1, delay=-1
  2. 创建 HowlingSuppression 检测模块（hc_mode=-2, hs_level=-1 仅检测不做陷波）
  3. 如果 dse_enable 已开启，启用 AINS 后处理模式（SetPostProcess=1）
  4. 初始化所有状态变量为 0

7. 模块互斥关系

组合行为宏控制
hs_enable + dse_enable共存：啸叫检测 + AINS 联合AEC_CONVERGENCE_ENHANCED
hs_enable + dse_enable互斥：DSE 开启时关闭 HS无宏（旧逻辑）
VOICE_ENHANCE + hs_enable互斥：人声增强关闭 HS—
dse_enable + ns_enable共存但 NS 可被 skip—
hpf_enable + dse_enable互斥：DSE 开启时跳过 HPF—
