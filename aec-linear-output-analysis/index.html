AEC 线性输出频谱条纹失真分析

为什么时域相减会产生频谱横向条纹？如何正确提取线性 AEC 输出？

2026年5月

1. 问题现象

在 WebRTC AEC 中，自适应滤波器的线性输出 e[i] = nearend[i] - y[i] 直接保存为 PCM 文件后，频谱图中出现明显的等间距横向条纹（水平方向的周期性能量分布）。

而经过 NLP（非线性处理）后的输出则频谱干净、平滑，没有这种条纹。

核心问题：为什么简单的时域相减 e = nearend - y 会产生频谱失真？

2. 信号处理原理分析

2.1 Overlap-Save 结构中的时域相减问题

WebRTC AEC 使用 128 点 FFT（PART_LEN2=128），每次处理 64 点新数据（PART_LEN=64）。回声估计 y[i] 的计算过程是：

  1. 远端信号经过频域自适应滤波：Y(k) = X(k) · W(k)
  2. IFFT 得到 128 点时域信号
  3. 取后 64 点作为有效输出：y[i] = ifft[PART_LEN + i] × scale
关键问题：y[i] 是通过 overlap-save 方法从频域滤波得到的，它只是 128 点 IFFT 输出的后半段。当自适应滤波器系数不完美时（未完全收敛、非线性失真、延时偏差），y[i] 与真实回声之间存在频率相关的误差。

2.2 频谱条纹的产生机制

横向条纹的本质是频域上等间距的能量凸起，产生原因有两个层面：

原因一：Block 边界不连续（主要原因）

每 64 点一个 block 直接拼接时，相邻 block 的边界处信号值可能不连续。这种周期性的不连续（每 64 点 = 4ms @16kHz）在频域表现为：

fstripe = fs / PART_LEN = 16000 / 64 = 250 Hz 及其谐波

频谱图上就会看到以 250Hz 为间隔的等距横向条纹。

原因二：自适应滤波器的频率栅格效应

128 点 FFT 的频率分辨率为 Δf = fs/N = 16000/128 = 125Hz。自适应滤波器在频域逐 bin 更新系数，相邻频率 bin 之间的系数可能存在不连续跳变。当滤波器未完全收敛时，某些频率 bin 的回声消除效果好，某些差，形成周期性的残余能量分布。

原因三：滤波器增益放大

自适应滤波器在某些频带可能产生增益放大（即 |E(k)| > |D(k)|，残差能量超过原始近端能量）。这在以下情况常见：

  • 滤波器刚开始收敛，系数不稳定
  • 双讲（near-end + far-end 同时存在）
  • 回声路径突变
  • 延时估计偏差导致滤波器对齐错误

增益放大的频带在频谱上表现为异常亮的横向条带。

2.3 为什么 NLP 输出没有条纹？

NLP 对线性输出做了三件关键的事情：

  1. WOLA 重构：加窗 → FFT → 处理 → IFFT → overlap-add，消除 block 边界不连续
  2. 频带能量限幅：if |E(k)|² > |D(k)|² then E(k) = D(k)，消除增益放大
  3. NLP 增益抑制：基于相干性的频域增益 hNl[k]，进一步压制残余回声

3. 正确提取线性输出的关键步骤

3.1 完整的 WOLA（Weighted Overlap-Add）重构

不能直接拼接 e[64]，必须经过完整的 WOLA 流程：

eBuf[128] (上一帧e[64] + 当前帧e[64]) │ ▼ Step 1: 加窗 (WindowData) windowed[k] = eBuf[k] × sqrtHanning[k] (k=0..63) windowed[64+k] = eBuf[64+k] × sqrtHanning[64-k] (k=0..63) │ ▼ Step 2: FFT (128点) EFW[k] = FFT(windowed) → 频域复数表示 │ ▼ Step 3: 频带能量限幅 ★关键★ if |EFW[k]|² > |DFW[k]|²: EFW[k] = DFW[k] // 防止增益放大 │ ▼ Step 4: IFFT (128点) time[128] = IFFT(EFW) │ ▼ Step 5: Overlap-Add output[k] = time[k] × sqrtHanning[k] + outBuf[k] outBuf[k] = time[64+k] × sqrtHanning[64-k] // 存给下一帧

3.2 各步骤解决的具体问题

步骤解决的问题原理
加窗 (sqrtHanning)Block 边界不连续窗函数将信号两端压向零，消除边界跳变
FFT → IFFT帧间相位连续性频域变换保证 128 点信号内部的频率一致性，使前后半帧具有正确的相位关系
频带能量限幅滤波器增益放大确保残差不超过原始信号，消除异常频带的能量凸起
Overlap-Add帧间平滑过渡sqrtHanning² 满足完美重构条件，相邻帧重叠部分能量互补
最关键的步骤是「频带能量限幅」：即使做了完整的 WOLA 重构，如果不限制 |E(k)| ≤ |D(k)|，自适应滤波器在未收敛频带产生的增益放大仍然会表现为频谱条纹。这一步是消除条纹的决定性因素。

3.3 数学解释：为什么 sqrtHanning 能实现完美重构

sqrtHanning 窗满足 Princen-Bradley 条件：

w²[k] + w²[k + N/2] = 1,    对所有 k ∈ [0, N/2-1]

其中 N=128, N/2=64。这意味着相邻两帧的重叠部分加权后能量之和恒为 1，实现无失真的信号重构。

具体来说，对于第 n 帧和第 n+1 帧在重叠区域的重构：

output[k] = frame_n[64+k] × w[64-k] × w[k] + frame_{n+1}[k] × w[k] × w[k]

由于 w[k] × w[64-k] 的互补性，重叠区域的总增益恒为 1。

4. 最终代码实现

#ifdef AEC_DELAY_DUMP
if (aec->fp_linear_out) {
    float linear_fft[PART_LEN2];
    float linear_efw[2][PART_LEN1];
    float linear_dfw[2][PART_LEN1];
    short linear_out_buf[PART_LEN];
    float scale_l = 2.0f / PART_LEN2;
    int k;

    // Step 1: 对 eBuf 加窗 → efw
    for (k = 0; k < PART_LEN; k++) {
        linear_fft[k] = aec->eBuf[k] * EvzAec_sqrtHanning[k];
        linear_fft[PART_LEN + k] = aec->eBuf[PART_LEN + k]
                                   * EvzAec_sqrtHanning[PART_LEN - k];
    }
    aec_rdft_forward_128(linear_fft);
    // StoreAsComplex
    linear_efw[0][0] = linear_fft[0]; linear_efw[1][0] = 0;
    linear_efw[0][PART_LEN] = linear_fft[1]; linear_efw[1][PART_LEN] = 0;
    for (k = 1; k < PART_LEN; k++) {
        linear_efw[0][k] = linear_fft[2*k];
        linear_efw[1][k] = linear_fft[2*k+1];
    }

    // Step 2: 对 dBuf 加窗 → dfw
    for (k = 0; k < PART_LEN; k++) {
        linear_fft[k] = aec->dBuf[k] * EvzAec_sqrtHanning[k];
        linear_fft[PART_LEN + k] = aec->dBuf[PART_LEN + k]
                                   * EvzAec_sqrtHanning[PART_LEN - k];
    }
    aec_rdft_forward_128(linear_fft);
    linear_dfw[0][0] = linear_fft[0]; linear_dfw[1][0] = 0;
    linear_dfw[0][PART_LEN] = linear_fft[1]; linear_dfw[1][PART_LEN] = 0;
    for (k = 1; k < PART_LEN; k++) {
        linear_dfw[0][k] = linear_fft[2*k];
        linear_dfw[1][k] = linear_fft[2*k+1];
    }

    // Step 3: 频带能量限幅 ★关键★
    for (k = 0; k < PART_LEN1; k++) {
        if ((linear_efw[0][k]*linear_efw[0][k] + linear_efw[1][k]*linear_efw[1][k]) >
            (linear_dfw[0][k]*linear_dfw[0][k] + linear_dfw[1][k]*linear_dfw[1][k])) {
            linear_efw[0][k] = linear_dfw[0][k];
            linear_efw[1][k] = linear_dfw[1][k];
        }
    }

    // Step 4: IFFT
    linear_fft[0] = linear_efw[0][0];
    linear_fft[1] = linear_efw[0][PART_LEN];
    for (k = 1; k < PART_LEN; k++) {
        linear_fft[2*k] = linear_efw[0][k];
        linear_fft[2*k+1] = -linear_efw[1][k];
    }
    aec_rdft_inverse_128(linear_fft);

    // Step 5: Overlap-Add
    for (k = 0; k < PART_LEN; k++) {
        linear_fft[k] *= scale_l;
        float val = linear_fft[k] * EvzAec_sqrtHanning[k]
                  + aec->outBufLinear[k];
        linear_fft[PART_LEN+k] *= scale_l;
        aec->outBufLinear[k] = linear_fft[PART_LEN+k]
                             * EvzAec_sqrtHanning[PART_LEN-k];
        // 饱和输出
        if (val > 32767.f) val = 32767.f;
        else if (val < -32768.f) val = -32768.f;
        linear_out_buf[k] = (short)val;
    }
    fwrite(linear_out_buf, sizeof(short), PART_LEN, aec->fp_linear_out);
}
#endif

5. 总结

问题原因解决方法
频谱等距横向条纹Block 边界不连续 + 滤波器增益放大WOLA 重构 + 频带能量限幅
Block 边界不连续64 点直接拼接，相邻 block 相位跳变sqrtHanning 加窗 + overlap-add
滤波器增益放大未收敛频带的 |E(k)| > |D(k)|频带能量限幅：E(k) = min(E(k), D(k))
频率栅格效应128 点 FFT 分辨率 125Hz，逐 bin 系数不连续WOLA 的 FFT→IFFT 保证频域一致性
核心结论：在分块频域自适应滤波系统中，时域残差 e[n] = d[n] - y[n] 不能直接作为最终输出使用。必须经过 WOLA 重构（消除 block 边界效应）和频带能量限幅（消除滤波器增益放大），才能得到频谱干净的线性 AEC 输出。
